## 🎯 项目目标

- ✅ 直观的 Web 界面配置和管理 Agent
- ✅ 多种工作流模式（流水线/并行/讨论/混合）
- ✅ 实时监控和进度可视化
- ✅ Agent 间通信协作
- ✅ 手动干预功能
- ✅ 真实调用 OpenClaw 执行任务

## 📝 重要教训

1. **Skill 权限限制** → 采用独立服务器
2. **端口管理** → 不杀 OpenClaw 进程，换端口
3. **CLI 参数** → openclaw agent 需要 --session-id
4. **真实执行** → 独立服务器方案更灵活可控
